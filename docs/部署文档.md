# 生产任务排程管理系统 - 部署文档

## 1. 系统要求

### 1.1 硬件要求
| 组件 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 4核 Intel i5 | 8核 Intel Xeon E5 |
| 内存 | 8GB | 16GB+ |
| 硬盘 | 50GB SSD | 100GB SSD+ |
| 网络 | 100Mbps | 1000Mbps |

### 1.2 软件要求
| 软件 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | >= 20.0.0 | 建议使用 LTS 版本 |
| MySQL | >= 8.0 | 需启用 InnoDB 引擎 |
| Nginx | >= 1.18 | 生产环境用于反向代理 |
| 操作系统 | Windows Server 2016+/CentOS 7+/Ubuntu 20.04+ | 支持主流服务器操作系统 |

## 3. 数据库部署

### 3.1 安装 MySQL 8.0

**Windows 环境：**
1. 下载 MySQL 8.0 安装包：https://dev.mysql.com/downloads/mysql/
2. 运行安装程序，选择 "Server only" 或 "Full"
3. 配置端口：3306
4. 设置 root 密码：`password`（开发环境）
5. 安装完成后启动 MySQL 服务

**Linux 环境（Ubuntu/Debian）：**
```bash
sudo apt update
sudo apt install mysql-server -y
sudo mysql_secure_installation
sudo systemctl start mysql
sudo systemctl enable mysql
```

**Linux 环境（CentOS/RHEL）：**
```bash
sudo yum install mysql-server -y
sudo systemctl start mysqld
sudo systemctl enable mysqld
sudo grep 'temporary password' /var/log/mysqld.log
sudo mysql_secure_installation
```

### 3.2 配置 MySQL

编辑 MySQL 配置文件（Windows: my.ini, Linux: my.cnf）：
```ini
[mysqld]
# 基础配置
port = 3306
bind-address = 0.0.0.0
max_connections = 200

# InnoDB 配置（关键！）
default-storage-engine = INNODB
innodb_buffer_pool_size = 4G
innodb_log_file_size = 1G
innodb_flush_log_at_trx_commit = 1
innodb_lock_wait_timeout = 50
transaction-isolation = READ-COMMITTED

# 字符集
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci

# 时区
default-time-zone = '+08:00'

# 慢查询日志
slow_query_log = ON
long_query_time = 2
```

重启 MySQL 服务使配置生效。

### 3.3 初始化数据库

**方法一：使用命令行**
```bash
# Windows
mysql -u root -p < migrations\001_init_database.sql

# Linux
mysql -u root -p < migrations/001_init_database.sql
```

**方法二：使用 MySQL 客户端**
1. 打开 MySQL Workbench 或 Navicat
2. 连接到 MySQL 服务器
3. 执行 `migrations/001_init_database.sql` 脚本

**验证数据库：**
```sql
USE production_scheduling;
SHOW TABLES;

-- 应该看到以下表：
-- users, products, machines, orders, schedules, operation_logs

-- 验证初始数据
SELECT * FROM users;          -- 4条用户记录
SELECT * FROM products;       -- 5条产品记录
SELECT * FROM machines;       -- 6条机器记录
SELECT * FROM orders;         -- 5条订单记录
```

### 3.4 创建数据库用户（生产环境）
```sql
CREATE USER 'ps_user'@'%' IDENTIFIED BY 'YourStrongPassword123!';
GRANT SELECT, INSERT, UPDATE, DELETE, EXECUTE ON production_scheduling.* TO 'ps_user'@'%';
FLUSH PRIVILEGES;
```

## 4. 后端服务部署

### 4.1 安装 Node.js

**Windows 环境：**
1. 下载 Node.js 20 LTS：https://nodejs.org/
2. 运行安装程序，默认选项即可
3. 验证安装：`node --version && npm --version`

**Linux 环境：**
```bash
# 使用 NodeSource 安装
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install nodejs -y
node --version && npm --version
```

### 4.2 部署后端服务

1. **上传项目文件**
   将项目代码上传到服务器目录，例如：`/opt/production-scheduling`

2. **安装依赖**
   ```bash
   cd /opt/production-scheduling
   npm install --production
   ```

3. **配置环境变量**
   复制 `.env` 文件并根据实际环境修改：
   ```bash
   cp .env .env.production
   ```
   
   编辑 `.env.production`：
   ```env
   PORT=3001
   NODE_ENV=production
   
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=ps_user
   DB_PASSWORD=YourStrongPassword123!
   DB_NAME=production_scheduling
   
   JWT_SECRET=YourSuperSecretJWTKeyHere_MakeItLongAndRandom123!
   JWT_EXPIRES_IN=12h
   ```

4. **编译 TypeScript**
   ```bash
   npm run build
   ```

5. **使用 PM2 管理进程（推荐）**
   ```bash
   # 安装 PM2
   npm install pm2 -g
   
   # 启动服务
   pm2 start dist/api/index.js --name ps-api --env production
   
   # 设置开机自启
   pm2 startup
   pm2 save
   
   # 查看状态
   pm2 status
   pm2 logs ps-api
   ```

6. **验证后端服务**
   ```bash
   curl http://localhost:3001/api/auth/login \
     -X POST \
     -H "Content-Type: application/json" \
     -d '{"username":"admin","password":"password"}'
   ```
   
   应该返回：
   ```json
   {
     "code": 200,
     "message": "success",
     "data": {
       "token": "eyJhbGciOiJIUzI1NiIs...",
       "user": {...}
     },
     "timestamp": 1718000000000
   }
   ```

## 5. 前端部署

### 5.1 构建前端
```bash
cd /opt/production-scheduling
npm run build
```

构建完成后，静态文件将在 `dist/` 目录下。

### 5.2 配置 Nginx

安装 Nginx：
```bash
# Ubuntu/Debian
sudo apt install nginx -y

# CentOS/RHEL
sudo yum install nginx -y
```

创建配置文件 `/etc/nginx/conf.d/production-scheduling.conf`：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL 配置
    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 前端静态资源
    location / {
        root /opt/production-scheduling/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
        
        # 缓存配置
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时配置
        proxy_connect_timeout 30s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml;
}
```

测试并重启 Nginx：
```bash
sudo nginx -t
sudo systemctl restart nginx
sudo systemctl enable nginx
```

## 6. 数据备份策略

### 6.1 自动备份脚本

创建备份脚本 `/opt/production-scheduling/scripts/backup.sh`（Linux）：
```bash
#!/bin/bash

BACKUP_DIR="/opt/production-scheduling/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="production_scheduling"
DB_USER="root"
DB_PASS="password"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u$DB_USER -p$DB_PASS --single-transaction --routines --triggers $DB_NAME > $BACKUP_DIR/${DB_NAME}_$DATE.sql

# 压缩备份
gzip $BACKUP_DIR/${DB_NAME}_$DATE.sql

# 保留最近30天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

echo "备份完成: $BACKUP_DIR/${DB_NAME}_$DATE.sql.gz"
```

**Windows 备份脚本** `scripts\backup.bat`：
```batch
@echo off
setlocal

set BACKUP_DIR=D:\production-scheduling\backups
set DATE=%date:~0,4%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%%time:~6,2%
set DB_NAME=production_scheduling
set DB_USER=root
set DB_PASS=password

if not exist %BACKUP_DIR% mkdir %BACKUP_DIR%

mysqldump -u%DB_USER% -p%DB_PASS% --single-transaction --routines --triggers %DB_NAME% > %BACKUP_DIR%\%DB_NAME%_%DATE%.sql

powershell -Command "Compress-Archive -Path '%BACKUP_DIR%\%DB_NAME%_%DATE%.sql' -DestinationPath '%BACKUP_DIR%\%DB_NAME%_%DATE%.zip'"
del %BACKUP_DIR%\%DB_NAME%_%DATE%.sql

forfiles /p %BACKUP_DIR% /m *.zip /d -30 /c "cmd /c del @path"

echo 备份完成: %BACKUP_DIR%\%DB_NAME%_%DATE%.zip
```

### 6.2 设置定时备份

**Linux crontab：**
```bash
# 编辑 crontab
crontab -e

# 每天凌晨2点执行备份
0 2 * * * /opt/production-scheduling/scripts/backup.sh >> /var/log/ps-backup.log 2>&1
```

**Windows 任务计划：**
1. 打开"任务计划程序"
2. 创建基本任务：名称 "生产排程数据库备份"
3. 触发器：每天 02:00
4. 操作：启动程序 `scripts\backup.bat`
5. 保存并测试

### 6.3 数据恢复
```bash
# 解压备份
gunzip production_scheduling_20240101_020000.sql.gz

# 恢复数据库
mysql -u root -p production_scheduling < production_scheduling_20240101_020000.sql
```

## 8. 常见问题排查

### 8.1 数据库连接失败
1. 检查 MySQL 服务是否运行
2. 检查防火墙是否开放 3306 端口
3. 验证 `.env` 中的数据库配置
4. 检查 MySQL 用户权限

### 8.2 后端服务无法启动
1. 查看 PM2 日志：`pm2 logs ps-api`
2. 检查端口 3001 是否被占用：`netstat -ano | findstr 3001`
3. 确认 Node.js 版本 >= 20
4. 重新安装依赖：`rm -rf node_modules && npm install`

### 8.3 甘特图加载缓慢
1. 检查浏览器控制台是否有错误
2. 确认 API 响应正常
3. 检查排程数据量，考虑分页加载
4. 优化数据库查询索引

### 8.4 排程冲突检测不生效
1. 确认 MySQL 使用 InnoDB 引擎
2. 检查事务隔离级别是否为 READ-COMMITTED
3. 查看数据库锁等待情况：`SHOW ENGINE INNODB STATUS;`
4. 检查 schedules 表的索引是否正确创建

## 9. 安全加固建议

1. **修改默认密码**：所有默认用户密码必须修改
2. **配置防火墙**：只开放必要端口（80, 443）
3. **启用 HTTPS**：使用 Let's Encrypt 免费证书
4. **定期更新**：及时更新系统和依赖包
5. **访问控制**：配置 IP 白名单限制后台访问
6. **审计日志**：定期审查操作日志
7. **备份验证**：定期测试备份数据的可恢复性
